# My Week — Build Guide

This is a complete Flutter project. You provide two source files (already here):
`lib/main.dart` and `pubspec.yaml`. The platform/build config (the `android/`
folder) is generated by Flutter's own tooling, so there's nothing fragile for
you to hand-edit.

Pick **one** of the two paths below.

---

## Path A — Build in the cloud (no tools on your computer)

Best if you don't want to install anything. You only need a free GitHub account.

1. Go to github.com and create a new **empty** repository, e.g. `my-week`.
2. Upload these files, keeping the folder structure:
   - `lib/main.dart`
   - `pubspec.yaml`
   - `.github/workflows/build-apk.yml`
   (On GitHub: "Add file" → "Upload files". You can drag the whole folder.)
3. The upload to the `main` branch triggers the build automatically. (Or open
   the **Actions** tab → "Build APK" → "Run workflow".)
4. Wait ~3–5 minutes. Open the **Actions** tab, click the finished run, and
   under **Artifacts** download `my-week-apk`.
5. Unzip it — inside is `app-release.apk`. Send that file to your phone and
   install it (see "Install on your phone" below).

---

## Path B — Build on your own computer

Best if you'd rather build locally.

1. Install Flutter: https://docs.flutter.dev/get-started/install (follow the
   Android setup steps so `flutter doctor` shows no Android errors).
2. In a terminal, create a fresh project and drop in the two files:
   ```bash
   flutter create my_week
   cd my_week
   # Replace the two generated files with the ones from this folder:
   #   - copy lib/main.dart  over  my_week/lib/main.dart
   #   - copy pubspec.yaml   over  my_week/pubspec.yaml
   flutter pub get
   flutter build apk --release
   ```
3. The APK is at:
   `build/app/outputs/flutter-apk/app-release.apk`
4. Copy it to your phone and install it.

> If the release build ever complains about signing, run
> `flutter build apk --debug` instead — that APK also installs fine for personal use.

---

## Install on your phone

1. Transfer `app-release.apk` to the phone (USB, Google Drive, email, etc.).
2. Tap the file. Android will ask to allow installing from this source —
   enable it for your file manager / browser, then confirm.
3. Open **My Week**.

This is a personal/sideloaded build (not from the Play Store), so Android shows
the usual "unknown source" prompt. That's expected.

---

## What the app does

- One screen listing **Monday → Sunday** (weekends included, highlighted).
- **+ Task** to add a task: title, which day, a category, and an optional time.
- Tap a task to edit it; **swipe left** to delete it; tap the checkbox to mark done.
- **Categories** (tag icon, top right): create your own categories with a colour,
  and delete ones you no longer use.
- Everything is saved on the device and is there next time you open the app.
